Hierarchical data:

The goal of this study is to examine the effectiveness of existing comparison tools on visualizing changes in hierarchical data. Thus first we provide an overview of hierarchical data and visualization types for these data and then we explain three types of changes that we use to modify the hierarchical data named as "move", " add" and "delete" operation. And at last we will provide an explanation on comaprison types that have been used in this study.

Hirarchical data is defined as a set of entities that are related to each other by ranking principle. A hierarchy has a root entity that all other entities are descendents of this root. A root has at least one child entity and other child entities can have at least one or zero child.

An entity which comes below another is a child node to the entity above. Similarly, an entity which comes above another is a parent node to the node below. For example: in an employee hierarchy a CEO can be referred to as the root of all the employees which have at least one child entity as a manager (e.g., project manager,..)

Hierarchical data can be shown using trees, treemaps and sunburst diagrams.

Treemap:

Treemap visualizes entities in the form of nested rectangles. Each branch (parent node and its children) is given a rectangle, which is then tiled with smaller rectangles representing sub-branches (children). The area of the rectangles are defined as the data that represent each entity. In our design, hierarchical data is only represented with nodes and their relationship with parents or children.
